# reports/ 폴더 사용법

`build_dashboard.py`가 이 폴더의 모든 `.md` 파일을 읽어 `../index.html`로 변환합니다.
(이 README.md 파일 자체는 리포트로 처리되지 않습니다.)

## 파일 작성 방법

프런트매터 없이 그냥 마크다운만 써도 됩니다. 제목/날짜는 아래 순서로 자동 추출됩니다.

- 제목: 파일의 첫 번째 `# 헤딩`
- 날짜: 본문의 "2026년 07월 01일" / "생성일시: 2026-05-07" 같은 표기,
  또는 파일명의 `YYYY-Wnn`(예: `2026-W27.md`) / `YYYY-MM-DD` 패턴,
  둘 다 없으면 파일 수정시각을 사용

더 정확하게 제어하고 싶으면 파일 맨 위에 프런트매터를 추가하세요.

```markdown
---
title: 삼성전자 실적 분석
date: 2026-07-10
tags: [반도체, 실적]
summary: 3분기 영업이익 컨센서스 상회, 메모리 가격 반등 본격화
---

# 본문 시작...
```

## 대시보드 생성

```bash
python3 ../build_dashboard.py
```

`reports/` 폴더에 파일을 추가하거나 수정한 뒤에는 항상 위 명령을 다시 실행해야
`index.html`에 반영됩니다.
